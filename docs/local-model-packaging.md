# 本地模型与打包策略

PhysicsLab AI 的助教分为两层：

1. 基础助教：内置知识库、公式工具、当前仿真状态和学习记忆。无需 Ollama，网页打开即可工作。
2. 本地模型增强：连接当前设备的 Ollama 服务 `http://127.0.0.1:11434`，使用 `qwen3:4b` 增强复杂问答。

## 分享网页时会发生什么

如果把网站部署到 GitHub Pages 或其他静态站点，访问者的浏览器会尝试连接访问者自己电脑上的：

```text
http://127.0.0.1:11434
```

这不会自动连接开发者电脑上的 Ollama。也就是说：

- 对方没有安装 Ollama：基础助教仍可用，模型增强不可用。
- 对方安装了 Ollama 但没有模型：需要执行 `ollama pull qwen3:4b`。
- 对方安装了 Ollama 且有 `qwen3:4b`：模型增强可用。

## 不建议让别人调用开发者电脑的 Ollama

可以通过网络暴露 Ollama 服务，但不建议作为公开发布方案：

- 会把本机模型服务暴露到网络。
- 需要处理鉴权、限流、日志、滥用和跨域。
- 你的电脑要一直在线，性能和带宽都会成为瓶颈。
- 学生问题会发到你的电脑，涉及隐私和合规。

如果需要多人共享模型，应使用受控后端服务，而不是直接暴露本机 Ollama。

## 推荐发布路线

### 阶段 1：静态网页

发布网站本体，内置基础助教能力。模型增强是可选项。

### 阶段 2：桌面软件

用 Tauri 或 Electron 打包。首次启动时检测：

```text
Ollama 是否运行
qwen3:4b 是否安装
```

未安装时给出安装引导。

### 阶段 3：捆绑模型

只有在明确接受安装包体积和许可证要求后再做。`qwen3:4b` 体积约数 GB，直接捆绑会显著增加安装包大小，并且需要分别处理 macOS、Windows、Linux 的运行环境。

## 当前代码策略

- 默认不依赖模型。
- 模型不可用时不阻塞基础助教。
- AI 面板会显示“基础助教可用”和“模型增强”状态。
- 模型增强只连接当前设备的本机 Ollama。
